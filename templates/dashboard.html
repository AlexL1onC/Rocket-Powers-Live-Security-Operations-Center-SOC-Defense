Monitoreo en tiempo real de anomalías en tráfico de LLMs. La vista principal está enfocada en alertas de seguridad, mientras que las anomalías operativas se conservan como contexto forense.

Enfoque principal del dashboard:
En la vista de seguridad, el objetivo no es analizar latencia o tokens, sino identificar eventos con señales de riesgo: HTTP 401 por autenticación fallida, HTTP 403 por acceso prohibido, HTTP 429 por rate limit o posible abuso, y prompts con contenido sospechoso. Las gráficas de latencia y tokens se conservan como contexto operativo secundario.

Filtros de investigación SOC

Recomendación para demo: usar Solo seguridad. Si no aparecen alertas en la ventana seleccionada, cambia a Todas para mostrar el contexto forense.
Cómo leer este dashboard:
1. Las líneas muestran el comportamiento real observado en los logs almacenados en SAP HANA.
2. Los puntos muestran anomalías detectadas por el modelo ML según el filtro seleccionado.
3. El filtro Solo seguridad muestra únicamente anomalías que cumplen criterios de seguridad.
4. El filtro Solo operativas muestra anomalías de rendimiento o confiabilidad que no se escalan a la API de seguridad.

3. Vista principal de seguridad

Eventos de seguridad por tiempo

Esta gráfica muestra cuántos eventos fueron clasificados como seguridad por ventana de tiempo. Incluye accesos no autorizados, rate limit o patrones de abuso.

Distribución de códigos HTTP de seguridad

HTTP 401 y 403 indican posibles problemas de autenticación o autorización. HTTP 429 indica rate limit o posible abuso volumétrico.

4. Contexto operativo: datos reales vs anomalías

Nota sobre eventos de seguridad:
En la vista de seguridad, algunos eventos pueden aparecer con tokens y latencia en cero porque fueron bloqueados antes de llegar al modelo LLM. Para estos casos, la evidencia principal es el código HTTP, la IP, el servicio y la frecuencia del evento.

Tendencia real de tokens vs anomalías

6. Vista forense: puntos anómalos principales

Interpretación de la tabla: La columna Tipo indica si el evento es operativo o de seguridad. La columna Evidencia security muestra la razón por la que una anomalía puede ser considerada de seguridad. La columna Contexto operativo muestra señales complementarias como timeout, latencia alta o tokens elevados. Un estado success o timeout no se considera seguridad por sí solo.
